# GPF-001: Document the service onboarding loop

## PURPOSE

Create a reusable onboarding playbook that new service owners can follow from first access request through production readiness.

## CURRENT_STATE

- New owners currently rely on ad hoc Slack guidance.
- Access steps are spread across several internal docs.
- No single walkthrough explains the end-to-end onboarding loop.

## REQUIREMENTS

- Produce one source-of-truth playbook.
- Cover access, local setup, deployment expectations, and escalation paths.
- Keep the artifact generic enough to reuse across multiple teams.
