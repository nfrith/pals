### ALS-116
- Compatibility: refresh_required, additive
- Summary: Module-versioned `ignored_directories: string[]` lands as a new top-level peer of `dependencies`, `delamains`, and `entities` on the `module.ts` authored shape, so a module mounted at a parent path can now explicitly carve out subdirectories that ALS validation skips. The contract is deliberately narrow: entries are normalized relative directory paths under the mount root (single-segment like `doctrine` or multi-segment like `meta/drafts`), and per-file exceptions, glob syntax, and inferred carve-outs stay rejected — SDR-003's all-or-nothing subtree-ownership rule is refined into "owned but non-record-bearing for explicitly declared directories," not relaxed. `nfrith-repos/als/alsc/compiler/src/schema.ts` extends `moduleShapeSchema` with an optional `ignored_directories` array, validates each entry against an `ignoredDirectoryPath` slug-segment regex (rejecting absolute paths, `.` / `..` segments, hidden segments, and trailing slashes), and `superRefine`s the list to reject duplicate entries, ancestor-descendant overlap via the existing `modulePathsOverlap` helper, and entries that could contain records for any declared entity path template via a new `ignoredDirectoryCanContainEntityRecords` helper that walks placeholder-aware template segments. `nfrith-repos/als/alsc/compiler/src/validate.ts` carries `ignored_directories` into `LoadedModuleContext`, prunes declared ignored subtrees during `discoverRecordFiles()` while still descending them via the new `collectIgnoredRecordLikeFiles()` helper so summary deltas count ignored record-like files exactly, and emits a `warning`-severity `SHAPE_CONTRACT_INVALID` diagnostic with the new machine-readable reason slug `module_shape.ignored_directory.missing` when a declared ignored directory does not exist on disk (so authored drift stays visible without turning validation red). `nfrith-repos/als/alsc/compiler/src/diagnostics.ts` adds the `MODULE_IGNORED_DIRECTORY_MISSING` reason slug. Outside ignored subtrees SDR-003 strictness is unchanged: stray `.md` / `.jsonl` at the mount root or under any non-ignored directory still emits `PAL-RV-PARSE-003`, and reserved `AGENTS.md` / `CLAUDE.md` continue to be globally ignored independent of any module declaration. Reference docs at `nfrith-repos/als/skills/docs/references/shape-language.md` and `nfrith-repos/als/skills/docs/references/vocabulary.md` document the new field in the canonical `module.ts` example, the path normalization rules, the "owned but non-record-bearing" framing, the asymmetric missing-directory warning posture, and the Module Data Record context. Out of scope and deliberately untouched: file-glob style ignores (`*.md`, `**/draft*.md`), per-file allowlists, nested-module support inside ignored regions, migration tooling for existing systems splitting one module into a unified ignore-aware mount, the `module-owner.ts` ownership rule (ignored paths still resolve to the parent module), `als_version`, `SUPPORTED_ALS_VERSIONS`, any engine-managed construct `VERSION` bump, and any v5-to-v6 language-upgrade recipe.

- Summary: Split the bundled plugin-shipped hook config files into per-harness subdirectories and restore `alsc validate` to a single-purpose ALS-system validator.
