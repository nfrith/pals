- Compatibility: refresh_required, additive
- Summary: Ship the ALS language-upgrade toolchain — a compiler-owned `language-upgrade-recipe` contract plus a sibling runtime engine and operator skill, leaving `als_version: 1` and every existing skill (`/update`, `/change`, `/migrate`, `/validate`, etc.) untouched. A new sibling runtime package `alsc/upgrade-language/` provides hop-chain planning (refusing any `target < current` plan), preflight, checkpointed runtime state, recovery routing on declared error conditions, AskUserQuestion-mediated pause/resume around `operator-prompt` steps, structured telemetry, and unconditional `.als/`-only mutation enforcement via post-step `git diff` inspection in a disposable clone (with `mutation_boundary_violation` as the failure mode). The operator surface lands as the `/upgrade-language` skill at `skills/upgrade-language/SKILL.md`. SDR 037 (`Accepted`) is the normative contract; canonical docs land at `language-upgrades/CLAUDE.md` and `skills/docs/references/language-upgrades.md`. The permanent frozen-fixture archive seeds `language-upgrades/fixtures/v1/` from `reference-system` (`.als/` plus the six mounted module roots, no `.claude/`, 18 modules under `.als/modules/`). `alsc/compiler/src/index.ts` becomes a thin `validate`-prefixed shim so the legacy bare invocation `bun src/index.ts <system-root>` keeps routing to `validate` — REQ-11 preserved. No real `v1 → v2` recipe ships in this job (REQ-11 forbids it before ALS v2 exists); the engine ships with synthetic example recipes and a runner test suite (7 pass / 23 expects) plus 20 compiler tests covering the inspection contract. Real agent-task dispatch is left for the first language-version-hop job — today the runner returns `agent_executor_missing` for `agent-task` steps. Construct-upgrade work stays out of scope and rides on sibling [ALS-067](als-factory/jobs/ALS-067.md); the integration is a consumer relationship documented in SDR 037's Non-Goals.

No data migration; no authored-system changes; `als_version: 1` remains valid; no recipe runs in this release because no v2 hop exists yet. Existing `/update`, `/change`, `/migrate` flows continue unchanged.
