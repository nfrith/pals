# SWF-002: Add feature flag rollout dashboard

## DESCRIPTION

Operators need one place to see feature-flag rollout state, audience size, and rollback readiness before enabling a flag system-wide.

## CURRENT_STATE

- Flag state is split between an internal admin page and raw query output.
- Rollout ownership is unclear during incidents because nobody can quickly see who changed what.

## REQUIREMENTS

- Show flag state, current audience percentage, and last actor.
- Support rollback confirmation without leaving the dashboard.
- Keep the first pass read-heavy and operationally safe.

## DOD

- [x] Operators can inspect rollout state from one screen.
- [x] Rollback readiness is visible before expansion.
- [x] Audit context is present for the latest change.

## PLAN

### Scope

- Add a dashboard route backed by the existing flag service.
- Pull audit metadata into the summary cards.
- Add rollback-confirmation affordances only, not direct mutation.

### Verification

- Exercise one staged flag and one production flag.
- Confirm audit metadata renders even when no rollout is active.
